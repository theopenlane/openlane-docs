## Supported Standards

Openlane supports a variety of compliance standards, each with its own set of controls and requirements. These standards help organizations manage their security and compliance posture effectively. Below is an overview of the key standards available in Openlane:

- [SOC 2](soc2/overview): A standard for managing customer data based on five trust service criteria: security, availability, processing integrity, confidentiality, and privacy.
- [NIST Cybersecurity Framework (CSF)](nist/csf/overview): A framework for managing cybersecurity risk.
- [NIST 800-53](nist/80053/revision5): A catalog of security and privacy controls for federal information systems.
- [ISO 27001](iso27001/overview): An international standard for information security management systems (ISMS).
- Custom Standards: Organizations can create custom standards tailored to their specific compliance needs.

## Standard Properties

### Core Fields
- **Name**: Full name of the standard (e.g., "Service Organization Control 2")
- **Short Name**: Common abbreviation (e.g., "SOC 2")
- **Framework**: Unique identifier with version (e.g., "SOC2-2017")
- **Governing Body**: Organization that manages the standard (e.g., "AICPA")
- **Version**: Standard version number
- **Domains**: Applicable security domains (availability, confidentiality, etc.)
- **Status**: Current status (active, draft, archived)

### Access Control
- **Is Public**: Available to all users in the system
- **Free to Use**: Can be used without licensing restrictions
- **Organization**: Owner organization for private standards

### Documentation
- **Description**: Detailed description of the standard's purpose and scope
- **Details**: Implementation guidance and requirements
- **References**: External links and documentation
