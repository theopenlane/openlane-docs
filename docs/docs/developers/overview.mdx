# Developer Documentation

Use this section to find implementation and operational guidance for the Openlane platform.

## Section Map

1. [Getting Started](./getting-started/overview.mdx): local setup, stack startup, and CLI authentication
1. [Core Development](./core-development/overview.mdx): repository structure, schemas, GraphQL/codegen, workflows, and jobs
1. [Integrations](./integrations/overview.mdx): OAuth, SSO/OIDC, email, billing, and ingest patterns
1. [Security](./security/overview.mdx): authentication, token management, encryption, and impersonation controls
1. [Operations](./operations/overview.mdx): deployment, CI/CD, configuration, and runbooks
1. [Contributing](./contributing/overview.mdx): docs style guidance and reference material

## How To Use This Docs Set

1. Start with `Getting Started` for a local environment.
1. Move to `Core Development` when you are changing API, schema, or workflow behavior.
1. Use `Integrations`, `Security`, and `Operations` for implementation-specific workstreams.
1. Follow `Contributing` standards when updating docs.
